<objective>
Add pagination to the product listing page and implement debounced search to reduce API call frequency, all with the new color palette.

**Purpose:** The current product page loads ALL products at once — this is slow, expensive, and bad UX. Pagination limits each request to a manageable page size. Debounced search prevents an API call on every keystroke, reducing backend load and improving responsiveness.

**Output:**
1. Updated `useProductStore.js` with pagination fields and API params
2. Updated `ProductPage.jsx` with pagination UI, debounced search, and new palette
3. Updated `Search.jsx` with debounced search implementation
</objective>

- GET `Products/all` — returns ALL products (no pagination support assumed)
- GET `Products/search?query=...` — returns matching products
- GET `Products/category/:slug` — returns products by category

**Assumption**: The backend API supports `?page=1&limit=12` query parameters for pagination. If the backend doesn't support this, we implement client-side pagination as a fallback.

**Design decisions (per D-03, D-04, D-05):**
- Debounce delay: 300ms for search input
- Page size: 12 products per page (3 cols × 4 rows on desktop)
- Pagination style: Numbered page buttons with Prev/Next arrows
- Scroll to top on page change (for UX)
- Smart loader integration on the product page too
- Category filtering resets to page 1
- Search results can also be paginated if there are many results
